Workflow

From sentiment data to LoRA experiment report.

The project keeps the experiment intentionally small: SST-2 sentiment classification, a tiny BERT model, CPU training, and a repeatable pipeline for comparing full fine-tuning against LoRA adapters.

Pipeline

End-to-end experiment flow.

Each step maps to the repo artifacts: configuration, dataset loading, LoRA injection, training, evaluation, and report generation.

1. Config Experiment settings

configs/sst2_bert_tiny.yaml defines model, data, LoRA, and training options.

2. Data SST-2 subset

2,000 train examples and 500 eval examples keep runs CPU-friendly.

3. Base BERT-tiny

prajjwal1/bert-tiny provides a small encoder backbone.

4. Full FT Train all weights

Baseline updates all 4,386,178 parameters.

5. Custom LoRA Rank sweep

Runs r=2, r=4, r=8, and r=16 with frozen base weights.

6. PEFT Library check

PEFT LoRA validates the adapter behavior and checkpoint size.

7. Report Metrics and plots

Evaluation writes CSV metrics, plots, and a Markdown report.

Full fine-tuning path

The baseline lets every parameter receive gradient updates. This is simple and strong, but it creates one full model checkpoint per task and scales poorly when many specialized variants are needed.

Implementation map

What the repo actually does.

LoRA injection

src/lora.py wraps selected nn.Linear modules, freezes base weights, and exposes trainable adapter parameters.

Training modes

src/train.py runs either full fine-tuning or LoRA fine-tuning using the same data and evaluation settings.

Evaluation output

src/evaluate.py and scripts/make_report.py produce outputs/metrics.csv, plots, and results.md.

Scaled-down context: This is a mechanics-focused reproduction on BERT-tiny and a small SST-2 subset, not a full large-language-model reproduction of the original LoRA paper.
