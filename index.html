Lightweight LoRA reproduction

Fine-tuning fewer parameters without giving up accuracy.

This project reproduces the core mechanics of Low-Rank Adaptation on SST-2 sentiment classification using a CPU-friendly BERT-tiny setup. It compares full fine-tuning, a custom LoRA implementation, and Hugging Face PEFT.

What was built

A compact reproduction of LoRA mechanics.

Custom LoRA implementation

The repo adds LoRA wrappers around selected linear layers, freezes the base weights, trains low-rank matrices A and B, and supports merge and unmerge behavior for adapted weights.

  • Target modules include query, key, value, and dense layers.
  • Rank sweep covers r=2, r=4, r=8, and r=16.
  • Metrics track accuracy, F1, trainable parameters, time, and checkpoint size.

Why LoRA matters

Instead of rewriting every model weight, LoRA learns a compact task-specific update. The base model stays fixed, and small adapters can be trained, stored, swapped, or served independently.

  • Cheaper customization for domain-specific tasks.
  • Faster experimentation than full fine-tuning.
  • Practical deployment pattern for many users or customers.
